Kết luận

State Pattern: Giúp đơn hàng tự động thay đổi hành vi (kiểm tra -> đóng gói) khi chuyển trạng thái mà không cần dùng if-else phức tạp.

Strategy Pattern: Giúp tách biệt logic thanh toán, dễ dàng thêm phương thức thanh toán mới (ví dụ: Ví điện tử) mà không ảnh hưởng lớp Order.

Decorator Pattern: Cho phép thêm các dịch vụ giá trị gia tăng (bảo hiểm, gói quà) vào đơn hàng một cách linh hoạt tại runtime.
